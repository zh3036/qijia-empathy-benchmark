# 补充对话 · 文件格式

在 `submissions/` 下新建一个 `.md` 文件，按下面的结构填写（复制即可）：

```markdown
# good 情绪命名得当

- 倾向：明显好 ✅
- 来源：原创
- 为什么：一句话说明它在共情上明显好或差的原因。

---

**第 1 轮**

> **教练：** ……
> **来访者：** ……

**第 2 轮**

> **教练：** ……
> **来访者：** ……

（……继续）
```

要点：

- 标题：在 `good` / `bad` 里二选一并删掉多余的，再写一句话，例如 `# good 情绪命名得当` 或 `# bad 冷漠说教`。**不要**把 `EXAMPLE.md` 里的“示例·不计入”标签抄进来。
- 文件名同样以倾向开头：`good-xxx.md` / `bad-xxx.md`。
- 每句前写 **教练：** 或 **来访者：**（这两个标注是关键；“第 N 轮”可有可无）。
- 如果是从 `data/low-rating-pool/` 选的，对话正文可直接引用该文件，**来源处务必写明出处**。
- 真实对话**必须脱敏**后再提交。

完整示例见 [`submissions/EXAMPLE.md`](submissions/EXAMPLE.md)。
