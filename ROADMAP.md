# 路线图

本仓库分阶段建设。当前为第一阶段。

## 阶段一 · 最小可用（当前）✅

- 数据：共情量表、20 段评分对话、聚合 + 原始分数、109 段低分对话池。
- 任务脚手架：锚定示例模板（9 个题目）、补充对话模板。
- 贡献流程：基于 Pull Request 编辑结构化文件。

目标：让志愿者**现在就能开始**贡献，流程尽量简单。

## 阶段二 · 网页浏览器（计划中）

一个静态网页，把**对话 + 它的分数分布**并排展示，方便志愿者快速浏览、筛选高/低分片段，而不必在文件间来回跳转。可复用现有的 `tes_anchor_reference.html` 思路。

- 触发条件：第一阶段贡献跑起来后，若发现“翻文件找素材”成为瓶颈。

## 阶段三 · 提交校验工具（计划中）

脚本/CI，自动检查志愿者 PR 的格式：

- 锚定示例：格式正确、注明来源、分数在 1–7。
- 补充对话：不与现有 20 段或锚定示例**重复**、标注了好/差倾向。

- 触发条件：贡献量上来后，人工 review 格式变得费时。

---

> 阶段二、三是**有意延后**的——先用最小版本验证志愿者协作是否顺畅，再决定投入。欢迎在 Issue 里反馈你觉得最该先做哪一个。
